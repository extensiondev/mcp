`publish` turns a build into a **shareable [extension.dev](https://extension.dev) URL** you can hand to a tester, reviewer, or teammate.

<Note>
Building your extension is **local and free** — see [`build`](/docs/commands/build). `publish` is the one step that talks to the hosted platform, so it's the only place Extension.js asks for an account token.
</Note>

## How it relates to the rest of Extension.js

Everything else in Extension.js — dev, logs, inspect, act, the [agent bridge](/docs/ai/agent-bridge) — is local and needs no account. **Hosted distribution is the single paid/platform surface**, and `publish` is its entry point. The command is a thin wrapper: it posts to the platform and prints the URL the platform returns.

## Authentication

`publish` authenticates with **`EXTENSION_DEV_TOKEN`** — a workspace/project access token. Create one in the [extension.dev](https://extension.dev) dashboard and provide it via the environment or `--token`:

<CodeGroup>

```bash env var
EXTENSION_DEV_TOKEN=<token> extension publish
```

```bash flag
extension publish --token <token>
```

</CodeGroup>

<Note>
There is no `extension login` command today — authentication is the `EXTENSION_DEV_TOKEN` value, nothing more. (A streamlined login flow is planned; until it ships, use a dashboard-minted token.)
</Note>

## Usage

```bash
extension publish [path] [options]
```

## Flags

| Flag | What it does | Default |
| --- | --- | --- |
| `--token <token>` | extension.dev access token (or `EXTENSION_DEV_TOKEN`). | — |
| `--ttl <hours>` | Share-link lifetime, `1`–`168`. | `24` |
| `--build-sha <sha>` | Pin the share URL to a specific build. | latest |
| `--api <url>` | Platform base URL (or `EXTENSION_DEV_API_URL`). | `https://www.extension.dev` |
| `--output <pretty\|json>` | Output format. `json` prints the full platform response. | `pretty` |

## Examples

Print a shareable URL for the current project:

```bash
EXTENSION_DEV_TOKEN=<token> extension publish
```

Pin a specific build and keep the link alive for a week, as JSON for scripts:

```bash
EXTENSION_DEV_TOKEN=<token> extension publish \
  --build-sha 672c4f2 --ttl 168 --output json
```

`pretty` prints just the share URL; `json` returns the full response (`shareUrl`, `visibility`, and—when applicable—`expiresAt`).

## For AI assistants

With the [MCP server](/docs/ai/mcp), this is `extension_publish` — the **only** tool that talks to the hosted platform. It reads `EXTENSION_DEV_TOKEN` from the environment; the MCP server never holds the token itself.
